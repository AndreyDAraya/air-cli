# State Management Reference

## Concepts: Flows and Pulses

- **Flows** = reactive state values. Any widget reading a Flow inside `AirView` re-renders when it changes.
- **Pulses** = dispatchable actions (events) that can trigger logic in the state.

These are generated by `@GenerateState` via `air_generator` + `build_runner`.

## Defining State with @GenerateState

```dart
import 'package:air_framework/air_framework.dart';
part 'state.air.g.dart'; // ALWAYS add this part directive

@GenerateState('feature_name')  // the "namespace" for generated Flows/Pulses
class FeatureState extends _FeatureState {
  // Private fields → become StateFlows (AirController<T>)
  int _count = 0;
  bool _isLoading = false;
  String? _error;
  List<String> _items = [];

  // Public void/Future<void> methods → become Pulses
  @override
  void increment() {
    count++;          // setter generated by @GenerateState
  }

  @override
  Future<void> loadData() async {
    isLoading = true;
    try {
      items = await someService.fetchAll();
    } catch (e) {
      error = e.toString();
    } finally {
      isLoading = false;
    }
  }

  @override
  void reset() {
    count = 0;
    items = [];
    error = null;
  }
}
```

### Injecting Services into States

When working inside a class annotated with `@GenerateState`, you often need to access services registered in `AirDI`. The framework provides two ways to do this:

1. **Option 1: Using AirDI (Global access)**
   ```dart
   final someService = AirDI().get<SomeService>();
   ```

2. **Option 2: Using `inject<T>()` (Better & Preferred)**
   The `Air` framework provides a convenience method `inject<T>()` specifically for use within state classes.
   ```dart
   final someService = inject<SomeService>();
   ```

Run after changes:
```bash
dart run build_runner build --delete-conflicting-outputs
```

## Generated classes

After generation `@GenerateState('counter')` produces:
- `_CounterState` — base class with reactive setters
- `CounterFlows` — static accessors: `CounterFlows.count` → `AirController<int>`
- `CounterPulses` — static dispatchers: `CounterPulses.increment(null)`

Naming rule: `FeatureFlows.<fieldWithoutUnderscore>`, `FeaturePulses.<methodName>`.

## AirView — Automatic Dependency Tracking

```dart
// Wrap any sub-tree that needs to react to state
AirView((context) {
  // Any .value accessed here registers as a dependency
  final loading = CounterFlows.isLoading.value;
  if (loading) return const CircularProgressIndicator();

  return Column(children: [
    Text('Count: ${CounterFlows.count.value}'),
    ElevatedButton(
      onPressed: () => CounterPulses.increment(null),
      child: const Text('+'),
    ),
  ]);
});
```

AirView handles subscription teardown on dispose automatically.

## AirBuilder — Explicit single-key builder

When you only need a single state key:

```dart
AirBuilder<int>(
  stateKey: 'counter.count',
  initialValue: 0,
  builder: (context, value) => Text('Count: $value'),
);
```

For 2 keys use `AirBuilder2`, for 3 use `AirBuilder3`.

## TypedStateKey — Type-safe keys without @GenerateState

Use this when you don't need the full `@GenerateState` pipeline:

```dart
// Define once
const counterKey = SimpleStateKey<int>('counter', defaultValue: 0);

// Set value
Air().typedFlow(counterKey, 10);
counterKey.value = 42; // shorthand via extension

// Get value
final count = Air().typedGet(counterKey);
final count2 = counterKey.value; // shorthand

// Build UI
counterKey.build((context, value) => Text('$value'));
```

## Manual AirController (Low-level)

```dart
final myController = Air().state<String>('my.key', initialValue: '');
myController.value = 'hello';   // triggers rebuild
myController.update((v) => v + '!'); // transform
myController.setValue('hi', force: true); // bypass equality check
```
